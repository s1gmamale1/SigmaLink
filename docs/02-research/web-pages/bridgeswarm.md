## Feature list (verbatim from page)
- Coordinator (Staff Engineer/Tech Lead): decomposes goals into parallel tasks, assigns file ownership, manages dependencies, monitors progress.
- Builder (Senior Software Engineer): follows structured workflows, modifies only assigned files, matches code patterns, validates work before completion.
- Scout (Codebase Intelligence Specialist): maps codebase, provides structured intelligence on patterns/risks, eliminates discovery time.
- Reviewer (Principal Engineer / Quality Gate): reviews completed tasks against quality standards, checks correctness/security/consistency, blocks substandard work.
- File ownership: each task has exclusive ownership of files it modifies; no two concurrent tasks touch the same file; shared deps sequence automatically.
- Shared Source of Truth: orchestration layer keeps all agents in sync.
- Quality Gates: review pass/fail before merge.

## 8 behavioral rules (verbatim)
1. Agents always know full project context before starting.
2. Real-time status tracking keeps swarm synchronized.
3. Strict file ownership prevents merge conflicts by design.
4. Zero idle chatter—every message advances the goal.
5. Structured completion reporting prevents falling through cracks.
6. Automatic escalation when agents are blocked.
7. Safe git practices enforced at orchestration layer.
8. Agents prioritize shipping code over sending messages.
