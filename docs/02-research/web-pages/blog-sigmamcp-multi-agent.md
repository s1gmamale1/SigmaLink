## Feature list (verbatim)

- Task lifecycle tracking with explicit state changes.
- Parallel execution — split investigation, implementation, verification across agents.
- Reduced context drift; agents reference the same source of truth.

## Orchestration pattern (4 phases)
1. Plan — capture objective, constraints, acceptance criteria.
2. Execute — assign sub-tasks to specialized agents in parallel.
3. Verify — run lint, tests, risk checks before merge.
4. Persist — store findings in task knowledge for future work.
