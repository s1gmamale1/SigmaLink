## Agent role specs (4)
- Feature agents — build new functionality from spec.
- Review agents — audit for bugs, security, conventions.
- Test agents — generate and maintain test suites.
- Refactor agents — improve existing code without changing behavior.

## Review focus areas
- Implementation matches design.
- No unnecessary abstractions.
- Integrates with existing systems.
- Edge cases handled.

## Testing requirements
- Generate tests alongside features.
- Run tests automatically.
- Test boundaries (edge cases, error handling, integration).

## Deployment practices
- Feature flags for continuous deployment.
- Automated rollbacks.
- Canary environments first.

## Cultural traits
- Trust agents but verify output.
- Invest in prompt quality.
- Iterate fast and ship often.
