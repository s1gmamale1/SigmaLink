# SigmaLink — Bounded Contexts (DDD domain map)

Maintained per the `wrap-up` skill. Each context = one clear responsibility, its key modules, and its RPC namespace. The **moat** is per-pane git worktrees: every agent works in an isolated worktree, and the Orchestrator merges by conflict probability.

## Cross-cutting invariants
- **Worktree isolation per pane** — distinct `session.worktreePath`/`branch`; the Orchestrator merge order is computed from per-worktree `git.status` overlap.
- **RPC allowlist** — every renderer-invokable channel is in `shared/rpc-channels.ts` `CHANNELS`; a cross-ref test (`rpc-channels.test.ts`) enforces router↔allowlist parity. **A new channel is a QUINT** (router-shape + rpc-router + CHANNELS + the test's own `TYPED_ROUTER_CHANNELS` hand-list + `rpc/schemas.ts`) — the hand-list drifts in tandem with the allowlist (`workspaces.rename`/`openNew` were bridge-rejected for months; fixed #158).
- **`closed_at` is the durable deliberate-close axis (ADR-007)** — every close-lifecycle decision (toast suppression, tile rehydrate, live resume) keys off `agent_sessions.closed_at`, never `status` (racy: late onExit overwrites). All close sites route through `markPaneClosed` BEFORE `pty.kill`. **A new `agent_sessions` column must also join `core/sync/engine.ts` `COLUMN_ALLOWLIST`** or device sync silently drops it on remote apply — the `allowlist-drift.test.ts` detector only fires in FULL-suite runs (#161).
- **RPC output validation is fail-open (ARCH-9)** — `core/rpc/validate.ts` `validateChannelOutput` is **drift-detection only** (main is the trusted producer): a schema mismatch is logged but the response passes through unchanged — it must NEVER convert a working response into an error. Input validation (`validateChannelInput`) is the enforcing boundary.
- **CLAUDE.md as the agent-instruction boundary** — scope (C-5) + guardrails (C-9) are written as idempotent marker-delimited blocks into each worktree's CLAUDE.md (guidance-level; no settings/hook mutation).
- **Ruflo MCP transport is per-CLI STDIO; the HTTP daemon is GATED OFF (B4)** — the per-workspace Ruflo HTTP daemon (v1.6.0) is disabled by default via `ENABLE_RUFLO_HTTP_DAEMON=false` in `core/workspaces/factory.ts` because `@claude-flow/cli`/`ruflo` HTTP server-mode is upstream-broken in all versions (ignores `-t http` / crashes / the SF-14 install pin was unpublished). Panes get Ruflo via the SF-15 per-CLI stdio autowrite (`mcp-autowrite.ts` + `mcp-config-writer.ts`). Re-enable ONLY when a CLI version is verified to keep `mcp start -t http -p N` alive AND answer `GET /health`.
- **Provider MCP-config writers must be TOML-table-aware, not comment-marker-keyed (B1)** — codex (and any CLI that rewrites its own `config.toml`) strips/relocates comments, orphaning marker-pair regexes → duplicate tables pile up. The codex/browser writer + the ruflo autowrite share `src/main/lib/toml-merge.ts` (`findTomlTableRanges`/`replaceTomlTables`/`stripMarkerLines`): collapse `[mcp_servers.*]` tables BY NAME (idempotent fixpoint), never by marker.
- **`packages/voice-core/` is the live capture/transcription module** — the dead duplicates `global-capture`/`output-router` were removed v1.19.0, and the dead `core/voice/whisper-engine.ts` was removed in v2.0.0 (ARCH-6, grep-proven zero importers). BUT `core/voice/{model-registry,adapter,dispatcher,native-mac,native-win,diagnostics}` remain LIVE: `electron/main.ts` imports `core/voice/model-registry` (model download/lookup) and `rpc-router` imports the adapter/dispatcher/diagnostics. (`model-registry` was wrongly deleted as "dead" in v1.20.0 and RESTORED in v1.20.1 — it is NOT a dupe of the voice-core one; verify importers before touching.) See `reference_voice_core_dead_tree`.
- **Transcription engine is pluggable behind `WhisperEngine`** — `resolveTranscriptionEngine(kv['voice.transcriptionMode'])` returns local Whisper (default, `base.en-q5_1`) or the Gemini-CLI engine; failures fall back to local. Claude/Codex are dispatch targets only (no audio modality).
- **Voice engine has a SECOND consumer: the standalone SigmaVoice app** (repo `s1gmamale1/SigmaVoice`, macOS arm64 v0.3.2). It embeds SigmaLink as a git submodule and consumes `packages/voice-core` + `native/voice-*` via pnpm `link:` deps — `voice-core` stays the single source of truth across both. App source lives here under `sigma-voice/` (dev home; the repo is the release home, mirrored). The macOS AVAudioEngine tap installs with `format:nil` (NOT `outputFormatForBus:0`) to avoid a 44.1kHz/2ch format-mismatch abort — that fix also protects SigmaLink in-app voice. Open: W-SV1 (Windows `voice-whisper` MSVC link), W-SV2 (quit-time TSFN SIGABRT). See `reference_sigmavoice_cross_repo_native_gotchas`.
- **aidefence security boundary (H-19)** — `core/security/aidefence-gate.ts` is the opportunistic / local-first / never-fail-open gate: `scanInbound` (advisory), `scanIngested` (redact+annotate `read_files`/`search_memories`), `scrubOutbound` (final-reply PII). All Ruflo `aidefence_*` calls go through the shared `unwrapAidefence` (the live daemon returns the raw MCP `{content:[{text}]}` envelope, not a parsed verdict). The local PII redactor `core/security/pii-scrub.ts` (ReDoS-safe, E.164-anchored phone) is the single shared regex set consumed by BOTH the gate AND R-1's `core/remote/safety.ts`. Engine down ⇒ pass-through; a flagged scan never throws or hard-blocks the local operator.
