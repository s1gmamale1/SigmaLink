small/medium wishlist items + extracts the voice-capture stack into a
shared package with a standalone SigmaVoice app scaffold. Three

SigmaVoice → SigmaVoice extraction (groundwork)
------------------------------------------------

"SigmaVoice" was the original product name for SigmaLink's voice
module. This release extracts the self-contained global-capture stack
into a shared package so a standalone dictation app can consume it:

- @sigmalink/voice-core (new package, app/packages/voice-core/) —
  extracts global-capture + output-router + whisper-engine +
  model-registry with full dependency injection (no SigmaLink-
  specific imports).
- Native packages promoted to pnpm workspace members — voice-mac,
  voice-win, voice-whisper are now real workspace members instead of
  createRequire path-walking.
- SigmaLink consumes @sigmalink/voice-core (electron/main.ts) — no
  behavior change; all voice IPC channels + events work identically.
- @sigmalink/sigma-voice (new app scaffold, app/apps/sigma-voice/)
  — runnable standalone Electron app: Tray + global hotkey +
  buildGlobalCaptureController + minimal settings window. System-wide
  dictation, no workspace/pane/session logic.
- A1 hardware sample-rate detection — macOS native binding now
  reports the real hardware rate via onPcm; voice-core resampler uses
  it instead of hardcoded 48 kHz (fixes mild pitch error on 44.1 kHz
  hardware).

Deferred (documented in app/apps/sigma-voice/README.md): production
packaging for the standalone app — its own electron-builder target,
macOS entitlements, codesigning, DMG/NSIS matrix, CI lane. The
standalone app is a runnable dev scaffold this release, not a signed
installer.


Ruflo HTTP daemon Settings UI (B2)
----------------------------------

v1.6.0 shipped the per-workspace Ruflo HTTP daemon but had no UI.
This adds a "Ruflo Daemon" section to Settings → Ruflo:
- RufloHttpDaemonSupervisor.list() returns all daemon handles.
- ruflo.daemonStatus(workspaceId?) (best-effort /health probe for
  connection counts) + ruflo.restartDaemon(workspaceId).
- Settings table: workspace / status / port / PID / uptime /
  connections + Restart button. Polls every 5 s.
