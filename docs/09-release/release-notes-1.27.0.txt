Fixed
-----

- H-6 — win32 shell-first sentinel mismatch. spawnLocalPty (wrap side) and
  registry.create (sentinel-watch side) disagreed on win32: the command was
  shell-wrapped and emitted the CLI-exit sentinel, but the registry watched it
  as 'direct' (no watcher) -> raw sentinel markers leaked into the terminal and
  onCliExited never fired. Extracted a single resolveEffectiveSpawnMode() used
  by both sides; win32 is now consistently 'direct' end-to-end (un-dogfooded
  win32 shell-first stays off). No POSIX change.

- H-9 — alt-command fallback dead in shell-first. The [command, ...altCommands]
  walk only advanced on a synchronous ENOENT (direct mode only); shell-first
  injected the bare command into a shell so a missing binary produced "command
  not found" output, never a throw, so the walk never tried the alt. Added the
  same synchronous binary pre-flight to the shell-first path -> the alt resolves
  in BOTH modes.

- H-10 — duplicate-pane spawn leaked a PTY. On a UNIQUE constraint violation
  (workspace_id, pane_index) the spawned PTY was logged + suppressed without
  being killed -> orphaned child process, no DB row, no kill path. Now kill +
  forget the orphaned PTY before suppressing.

- H-12 — provider list leaked the internal 'shell' sentinel. providers.list
  mapped the raw registry (the Settings -> Providers tab only filtered
  'legacy'), exposing 'shell' over RPC. Filtered at the source.

- H-13 — shutdownRouter fire-and-forgot the daemon stop. The per-workspace
  HTTP-daemon stopAll() (SIGTERM->5s drain->SIGKILL) and the Telegram-bridge
  stop were void-discarded, so the process could exit before they drained ->
  orphaned daemon processes. shutdownRouter is now async and AWAITS them
  (self-bounded, can't hang quit); the before-quit handler holds the quit until
  the async teardown settles, then re-quits.

- H-15 — git.diff silent truncation. A diff over the 16 MiB buffer was silently
  cut. GitDiff gained a `truncated` flag (additive; renderer ignores until
  updated) set on the maxBuffer-exceeded path.

Deferred — H-7 (transactional migrations)
------------------------------------------

Wrapping migrations in an outer db.transaction() is incompatible with this
codebase: migrations 0003/0006/0015/0018/… own their BEGIN/COMMIT, and a nested
BEGIN throws. Those are already self-atomic, and the runner records a migration
only after its up() succeeds (a throw short-circuits before the tracking-row
insert -> clean retry next boot). Fully centralizing transactions (stripping
every migration's own BEGIN/COMMIT so the runner owns one transaction) is a
separate, real-DB-tested refactor.

Remaining H-class: full H-19 (per-tool ingestion scanning: read_files /
open_url / browser scrape + the built-but-unwired outbound PII scrub), then
R-2 (native Cursor CLI provider). No schema migrations in v1.27.0.
