🚨 Critical Sigma-dispatch-pane invisible-pane hotfix
-----------------------------------------------------

If you used the Sigma Assistant on v1.4.0–v1.5.2 to spawn a pane via
"open codex as pane N" or similar, the pane was created on disk +
in the database + as a running CLI process, but the renderer's
sessions state and swarm state never learned about it. The pane was
invisible in the Command Room grid + the sidebar agent badge stayed
at the pre-spawn count.

Root cause: the `assistant:dispatch-echo` handler dispatched
`SET_ACTIVE_SESSION` + `SET_ROOM` to navigate the user TO the new
pane, but never dispatched `ADD_SESSIONS` + `UPSERT_SWARM` to actually
add the new pane to the renderer's state. The +Pane button correctly
dispatched all three; the Sigma path skipped two.

Fix: in `use-sigma-dispatch-echo.ts`, the handler now refreshes
swarms + sessions via `rpcSilent.panes.listForWorkspace(workspaceId)`
and `rpcSilent.swarms.list(workspaceId)` immediately after each
dispatch-echo event, and dispatches `ADD_SESSIONS` + per-swarm
`UPSERT_SWARM` for the refreshed state. Mirrors the v1.4.3 #02
boot-restore hydration pattern.

If you have orphan panes from v1.5.2 or earlier (running CLI
processes that don't appear in your Command Room), you can either:
- Close + reopen the workspace (boot restore will pick them up), OR
- Re-launch SigmaLink — the orphan panes will reattach via the
  existing v1.2.7 session-restore + v1.4.3 panes.listForWorkspace
  pipeline.

No data loss; the panes were always real, just invisible.

🚨 Critical v1.4.9 provider-install renderer regression
--------------------------------------------------------

Caught by the new `CHANGELOG`-vs-`AppRouter` defensive test on its
first run against current main. `providers.spawnInstall`,
`providers.setInstallConsent`, and `providers.getInstallConsent`
have been missing from the `CHANGELOG` allowlist in
`app/src/shared/rpc-channels.ts` since v1.4.9 PR #49 shipped.

The preload bridge hard-rejected the calls with "IPC channel not
allowed" errors. `ProvidersTab.tsx:84` silently swallowed via
`.catch(() => null)` — so the consent-state UI showed wrong values
without any user-facing error. `ProviderInstallModal.tsx:153,163,175`
raised sonner error toasts on user click.

Live broken since 2026-05-20 v1.4.9 release.

Fixed in this release: 3 channels added to `CHANGELOG`.

Internal-use distribution posture
---------------------------------

SigmaLink is currently developed for internal use only. Not selling,
not publicly distributing. The forward-looking "funded-only / EV
cert / WinGet / Microsoft Store / Apple Developer Program /
Picovoice licensing" framing has been removed from WISHLIST.md and
BACKLOG.md. Signed-distribution paths are explicitly OFF-ROADMAP.
The SmartScreen workaround in `app/build/nsis/README — First launch.txt`
and Gatekeeper ad-hoc-signing in `scripts/install-macos.sh` are
canonical for internal users. "Hey Sigma" wake-word options updated
to whisper.cpp continuous mode and OS-level dictation (no third-party
licensing needed).
