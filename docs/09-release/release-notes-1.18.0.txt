SigmaLink v1.18.0 — 2026-05-24
==============================

v1.18.0 — "Sigma Agent" (Milestone M3, the capstone of the
BridgeMind-competitive roadmap). One goal becomes N worktree-isolated agent
panes, each briefed with a plan capsule, with a conflict-aware merge order —
the "swarm with no merge conflicts" that a shared-directory competitor cannot
structurally match.

Compose-first: every primitive already shipped in M1/M2 + existing infra, so
M3 is mostly orchestration glue. Brainstorm -> code-verified plan -> 3 parallel
worktree-isolated coders, lead-merged with one combined hard gate in main.

Spec: docs/superpowers/specs/2026-05-22-bridgemind-competitive-roadmap-design.md


C-7 — Sigma Agent orchestrator
------------------------------

- A new OrchestratorPanel in the OperatorConsole "chat" tab. The operator
  authors N tasks (title / prompt / target files / success criteria /
  out-of-scope), then:
    1. "Launch swarm" -> rpc.swarms.create({ preset:'custom', roster:N })
       spawns N worktree-isolated panes in ONE call (each its own branch +
       worktree + mailbox).
    2. Each pane is auto-briefed via rpc.panes.brief — the M2 plan capsule +
       per-worktree CLAUDE.md scope block.
    3. "Propose merge order" reads each pane's changed-file set
       (rpc.git.status union) and orders panes by ascending pairwise overlap
       (fewest conflicts first).
    4. "Merge in order" runs rpc.review.batchCommitAndMerge on the sorted
       sessionIds (sequential --no-ff; stops on first conflict for manual
       resolution).
- RPC-free: composes existing channels only. Manual task authoring for v1;
  LLM goal auto-decomposition is a later layer.


C-10b — Inline voice into the focused pane
------------------------------------------

- Dictation can now land directly in the currently focused PTY pane instead of
  routing to the Jorvis assistant. A renderer->main push (voice:focused-session)
  tells the main process which pane is active; routeTranscript gains a
  focused-pane branch that pty.writes the (already dictionary/macro-normalized)
  transcript. Opt-in via a "Dictate into the focused pane" toggle in VoiceTab.
- True hold-to-record (vs the current toggle hotkey) is deferred — Electron's
  globalShortcut is keydown-only; a native hook lib is needed.


W-4 P8 — Resume simplification
------------------------------

- Removed the now-dead resumeMode ('id' | 'continue') label from the pane-resume
  result + toast. external_session_id is KEPT — exploration proved dropping it
  (the original P9) would regress multi-pane conversation resume for
  claude/codex/kimi/opencode (it is a pane->conversation binding, NOT redundant
  with shell-first PTY reattach). P9 cancelled by design.
