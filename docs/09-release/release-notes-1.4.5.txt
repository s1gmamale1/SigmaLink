SigmaLink v1.4.5 — Release Notes
================================

Date: 2026-05-18
Previous: v1.4.4 (2026-05-18)

This is a tech-debt cleanup release. No user-facing features.
Two reviewer followups closed and two long-standing file-size
debts retired.

What's fixed
------------

* gemini-resume-bridge's projects.json read-merge-write race is now
  fully serialized via proper-lockfile advisory locking. Two simultaneous
  pane spawns racing to register their worktree alias can no longer
  produce a missing entry. New concurrent-writers test verifies this.

* SessionStep test suite no longer flakes when run alongside the rest
  of the project. The remaining cross-suite module-state leak from
  v1.4.4 is fully closed by adding vi.resetModules() in beforeEach.
  Verified 5/5 in repeated full-suite runs.

Internal refactors
------------------

* swarms/factory.ts split: 443 LOC -> 271 LOC. addAgentToSwarm moved
  to a new factory-add-agent.ts sibling. Public API preserved.

* runClaudeCliTurn.ts split: 426 LOC -> 324 LOC. CLI arg + MCP + system
  prompt helpers moved to a new runClaudeCliTurn.args.ts sibling.
  Matches existing .emit.ts / .trajectory.ts pattern.

* React-compiler lint wave was investigated; turned out to be already
  closed by prior v1.1.9 work. No changes needed.

Dependencies
------------

* Added: proper-lockfile@^4.1.2 + @types/proper-lockfile@^4.1.4
