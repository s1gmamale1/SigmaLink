SigmaLink v1.4.0
================

Sigma Assistant now behaves like a resumable orchestrator instead of a fresh
one-shot Claude call each turn.

Highlights
----------

- Captures Claude's `system.init` session id for every Sigma Assistant
  conversation and stores it on the conversation row.
- Sends future turns in the same chat through `claude --resume <session-id>`.
- Retries once without resume when a stored session id is stale or missing on
  disk, clearing that id so the next turn starts cleanly.
- Adds a resumable indicator, right-rail conversation dropdown, resume banner,
  and interrupted-turn retry/dismiss banner in Sigma Assistant.
- Adds `assistant.conversations.resumeHint` for checking whether the stored
  Claude JSONL still exists for the conversation's workspace.

Developer Notes
---------------

- Adds migration `0013_conversations_claude_session_id`.
- Adds in-flight assistant message sentinel values shaped as
  `sigma-in-flight:<turnId>`.
