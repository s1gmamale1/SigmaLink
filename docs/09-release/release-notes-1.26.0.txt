SigmaLink v1.26.0 — 2026-05-26
==============================

Security hardening wave 1 — IPC containment (H-class backlog, wave 1 of 3).

Closes the "renderer trusts the path / no payload validation" attack surface in
the privileged Electron main process. The renderer is semi-trusted; a buggy or
compromised renderer is in the threat model. Built by 3 parallel worktree-
isolated Opus lanes -> lead-integrated the rpc-router wiring -> a MANDATORY Opus
security-review pass (verdict PASS, no Critical/High) -> fixes folded -> full
gate in main. No new dependencies, no schema migrations, no behavior change for
legitimate flows.


Security
--------

- Central path containment (H-5 / H-2 / H-3):
    * NEW core/security/path-guard.ts — assertAllowedPath(target, roots):
      resolves via realpath (symlink-safe), sep-boundary containment, THROWS
      'path outside workspace' on escape, fail-closed on empty roots.
    * One shared sandbox definition now used by the fs RPC controller,
      git.runCommand, pty.spawnScratch, pty.create, and the assistant
      read_files tool. Allowed roots = open workspaces' root/repo + the worktree
      pool, re-derived per call, deny-all on DB failure, no renderer influence.
    * fs.readDir / fs.readFile / fs.writeFile are now contained (were: any
      absolute path).
    * fs.writeFile NO LONGER trusts the renderer-supplied repoRoot (a
      repoRoot:"/" could previously collapse the path-traversal guard). repoRoot
      is deprecated; the allowed-roots provider is authoritative.
    * An in-tree symlink pointing outside the sandbox is rejected by realpath.

- Spawn-cwd containment (H-4):
    * git.runCommand, pty.spawnScratch, AND pty.create reject a renderer-
      supplied cwd outside the workspace/worktree sandbox before running or
      spawning. Commands already run shell:false (no shell-metachar injection);
      this bounds WHERE a process can spawn.

- IPC payload validation enforced (H-8):
    * NEW core/rpc/validate.ts — validateChannelInput parses each channel's
      input at the dispatch boundary; VALIDATION_MODE flipped to 'enforce'.
    * 17 path/command-carrying channels (fs.*, git.*, pty.*, kv.*,
      workspaces.*) gained concrete zod schemas (bounded strings, typed shapes)
      replacing z.any(). A malformed renderer payload is now rejected with an
      error envelope instead of reaching the controller. Unhardened channels
      stay z.any() (pass) and tighten incrementally.

- aidefence wired into the runtime (H-19, partial):
    * NEW core/security/aidefence-gate.ts — the previously-unused Ruflo
      aidefence engine now runs an opportunistic, NEVER-fail-open advisory scan
      on every assistant send prompt, emitting assistant:security events (flips
      Security: PENDING -> active). Local operator input is scanned + audited,
      never blocked; a Ruflo failure/absence can never break the assistant.
    * Remaining H-19 follow-up: per-tool ingestion (read_files / open_url /
      browser scrape) + outbound PII scrub (built + tested, not yet wired —
      no single egress chokepoint without touching the turn driver).


Fixed
-----

- H-11 — fs.readFile does a bounded partial read (open + read up to the cap),
  never reads the whole file into memory before truncating.
- H-16 — dirSize and getWorktreeSizes no longer follow symlinks (skip symlinked
  entries, lstat): no out-of-tree size traversal, no symlink-cycle recursion.

Security-review residuals (tracked, none blocking)
--------------------------------------------------

- fs.exists remains an existence oracle on arbitrary paths — it is legitimately
  used to probe system binary paths (e.g. /opt/homebrew/bin/<cli>) for
  provider-CLI install detection, so containing it would break that feature.
- git.runCommand's first token (the binary) is renderer-supplied; shell:false +
  the custom quote-only tokenizer mean NO shell-metachar injection, and the cwd
  is contained — residual is "can run a PATH binary inside a workspace dir".
- The side-band IPC handlers (swarm.*, swarm.replay.*, assistant.conversations.*,
  voice.diagnostics.*, sigmabench.*) don't yet route through
  validateChannelInput; those namespaces are still z.any(). Fold a shared
  ipcMain.handle wrapper when tightening them.
