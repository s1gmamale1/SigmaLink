C-12 â€” SigmaBench (multi-agent-conflict)
----------------------------------------

- A new room that runs the SAME task across N CLI providers (Claude / Codex /
  Gemini), each in its OWN isolated git worktree, then scores each provider by
  how much its changed-file set overlaps the others (lower = better isolation).
  This is the leaderboard a single-shared-directory competitor cannot produce.
- Reuses the C-7 merge-order overlap logic (`scoreConflicts`), `createSwarm`
  (N worktrees in one call), the 5-provider registry, and `gitStatus`.
- New: `benchmark_runs`/`benchmark_results` tables (migration 0023),
  `core/sigmabench/{harness,store}.ts`, `sigmabench.*` RPC, `SigmaBenchRoom`.
- MVP scope is the conflict category only; latency / code-quality categories
  are deferred. Running a bench spawns real CLI agents in throwaway worktrees.


C-13 â€” Element -> existing pane + diff

C-10c â€” CLI-based voice (local/cloud, "Both")
---------------------------------------------

- A Gemini-CLI transcription engine behind the WhisperEngine interface
  (audio -> WAV -> `gemini`), selectable via a VoiceTab "Transcription engine"
  toggle (Local Whisper / Gemini CLI).
- A "Send commands to" selector (Claude Code / Codex / Gemini) threaded into
  `routeTranscript` for where a voice command is dispatched.
- Claude Code & Codex are NOT transcription options (no audio modality); only
  Gemini can transcribe. Both remain valid dispatch targets.
- DEFAULTS UNCHANGED: transcription = local Whisper `base.en-q5_1`; dispatch =
  claude. The CLI paths are strictly opt-in (KV-gated), so nothing shells out
  until you toggle it.
- `wav-encode.ts`, `cli-transcribe-engine.ts`, `whisper-engine.ts`,
  `global-capture.ts`, `output-router.ts`, `VoiceTab.tsx`.
