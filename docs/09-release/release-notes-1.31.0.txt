SigmaLink v1.31.0 — 2026-05-27
==============================

SF-7 — Ruflo MCP auto-trust + health surfacing on workspace open.

A freshly-cloned repo opened as a SigmaLink workspace now connects Ruflo MCP
end-to-end: the bundled `ruflo` server is pre-approved per provider (no manual
/mcp trust accept), daemon health is visible in each pane's header, and the
previously-silent stdio fallback now surfaces.

SECURITY-SENSITIVE (it pre-approves an MCP server). Built by 2 worktree-
isolated lanes (Opus trust + Sonnet health dot) + a lead factory.ts
integration + a MANDATORY Opus security-review (verdict: PASS, no Critical/
High). Default-ON, opt-out, fail-open.


Added
-----

- Per-provider auto-trust of the bundled `ruflo` server ONLY
  (core/workspaces/mcp-trust.ts):
    * claude  -> additive, idempotent merge into
                 <root>/.claude/settings.local.json ->
                 enabledMcpjsonServers: ["ruflo"]. .local.json is gitignored,
                 so it doesn't dirty the clone. Pre-approves ONLY ruflo —
                 third-party servers in a cloned .mcp.json still prompt.
    * cursor  -> best-effort `cursor-agent mcp enable ruflo` (binary-gated,
                 args-array, fail-open; contract verified live).
    * codex / gemini / kimi / opencode -> verified no-ops (their MCP config
                 loads without a per-project trust prompt; the existing
                 mcp-autowrite config-write already means trusted).
  Gated on a new `ruflo.autoTrustMcp` KV (default ON) with a Settings -> Ruflo
  opt-out toggle ("pre-approves only SigmaLink's own ruflo server by name").

- Pane-header Ruflo health dot (useRufloDaemonHealth + PaneHeader.tsx),
  reusing the existing ruflo.daemonStatus RPC:
    green = running · amber = stdio-fallback · red = down · slate = unknown,
  with a tooltip. Polls via rpcSilent (never fires error toasts).

- stdio-fallback notification (ruflo-fallback-notice.ts): when the per-
  workspace HTTP daemon can't spawn (binary missing / port collision) and
  falls back to stdio, a one-time `info` notice surfaces it — previously a
  silent console.warn. Wired through a new OpenWorkspaceDeps.notifications
  sink (passed the NotificationsManager at the rpc-router open call site).


Security (review checklist — all PASS)
--------------------------------------

- Only the literal "ruflo" is ever trusted. Never enableAllProjectMcpServers,
  never a wildcard, never --dangerously-skip-permissions.
- claude merge is additive + non-destructive (other servers/keys preserved;
  unparseable or non-object settings file left byte-for-byte untouched).
- Atomic write with a UNIQUE temp-file name (pid+timestamp), with orphan-temp
  cleanup on rename failure — closes a theoretical concurrent-open TOCTOU.
- Fully fail-open: a workspace always opens even if trust/notify throws.
- cursor exec is binary-gated, uses an args array (no shell string), fail-open.
- No secrets logged.
