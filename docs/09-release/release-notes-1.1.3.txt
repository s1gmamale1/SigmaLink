SigmaLink v1.1.3 — Multi-workspace + pane resume + growable swarms

v1.1.2 made Sigma Assistant actually dispatch tools. v1.1.3 makes
SigmaLink behave like a real multi-project workspace: open several
projects in parallel like browser tabs, switch between them without
losing context, restart the app and find everything where you left it
— including the running CLI agents themselves.

What's new
==========

1. Multi-workspace tabs
----------------------

Sidebar gains real workspace tabs: 8 visible + overflow drawer beyond
that. Open as many workspaces as you want. Click between them — each
workspace's runtime context (sessions, swarms, Sigma conversation)
stays alive. Close button per tab dismisses just that one; a "+"
button opens the picker for adding more. On quit, every open tab is
persisted; on relaunch, all of them reappear with the correct active
workspace selected.

State model change: `activeWorkspace` (single) → `openWorkspaces[]` +
`activeWorkspaceId` selector pattern. Consumers read via the derived
selector so existing components (BridgeRoom, Composer, RightRail)
keep working without changes.

2. Pane resume across restart
-----------------------------

Each spawned agent CLI's session id is captured from its early
stdout output (Claude Code via JSONL `system.init` envelope, Codex
via banner regex, Gemini deferred until support confirmed) and
persisted to the new `external_session_id` column added in migration
0011. On the next app launch, the resume launcher batch-respawns
every restored workspace's panes with `<cli> --resume <session_id>`,
finally consuming the long-dormant `resumeArgs` registry field. Chat
history continues from where you quit.

Failures fall back to fresh spawns silently per the locked UX
decision; no prompts on restart.

3. Growable swarms (+ Sigma can do it too)
------------------------------------------

The swarm preset (5 / 10 / 15 / 20 panes) no longer locks the pane
count at create time. Command Room's top bar gains a "+ pane"
button; Swarm Room header gains "+ agent". Picker selects provider +
optional initial prompt. Capacity-bound at 20 (existing preset max).
GridLayout shapes extended to handle 13-20 panes (4×4 + 5×4).

Sigma Assistant can do the same via the new `add_agent` tool wired
through the MCP host server from v1.1.2. Ask Sigma "spawn one more
codex pane with a hello prompt" and watch it appear.

4. Ruflo pre-flight on workspace open
-------------------------------------

Workspace creation no longer leaves Ruflo dormant until you click
Download in Settings. `openWorkspace` now calls
`rufloSupervisor.ensureStarted()` (idempotent, non-blocking) +
`verifyForWorkspace(root, mode)`. Mode defaults to `fast` (~50ms
config file readback verifying the per-CLI MCP entries from the
v1.1.2 autowrite). A "Strict MCP verification" toggle in Settings →
Ruflo opts into the thorough `claude/codex/gemini mcp list` probe
(~3-5s) for paranoid users.

Status surfaces as a 3-state readiness pill in the workspace
breadcrumb area: green (verified), amber (partial — some CLIs
failed), red (Ruflo unavailable).

5. Per-CLI skills verification
------------------------------

When a workspace opens, `skillsManager.verifyFanoutForWorkspace()`
content-hash checks each enabled skill against each enabled CLI's
expected path (`~/.claude/skills/`, `~/.codex/skills/`,
`~/.gemini/extensions/sigmalink-*/`). Missing or stale skills
trigger silent re-fanout. The readiness pill aggregates skill
status alongside Ruflo status.

Distribution
============

arm64-only macOS DMG, same as v1.1.1 / v1.1.2. The build host's
native modules get packaged; per-arch CI rebuild matrix deferred to
v1.2. Intel-Mac users: stay on v1.0.1 or wait. Pre-build dance for
each manual arm64 release:

  npx electron-rebuild --module-dir \
    node_modules/.pnpm/better-sqlite3@<ver>/node_modules/better-sqlite3 \
    --types prod -f

Pin to Electron 30.5.1 (NMV 123) — the host's `node` 26 (NMV 147) is
binary-incompatible if you forget `--version` to electron-rebuild.

Upgrade path
============

Drop the new SigmaLink.app into /Applications. Existing kv state +
workspaces survive — migration 0011 is additive (adds nullable
column). On first launch after upgrading, your last-active workspace
opens automatically (legacy single-workspace session snapshot is
parsed as a fallback). Workspaces opened in this v1.1.3 session
forward will be restored as a list on subsequent restarts.
