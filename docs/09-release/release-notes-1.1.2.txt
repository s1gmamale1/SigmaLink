SigmaLink v1.1.2 — Sigma Assistant end-to-end

v1.1.1 wired the assistant to the local Claude Code CLI, but the Tool
calls panel exposed that `tool_use` envelopes never executed — Sigma
could talk, but it couldn't actually launch panes, dispatch prompts,
or see live workspace state. v1.1.2 closes that gap. After this
release Sigma is a fully-functional in-app assistant: it executes host
tools end-to-end, sees real-time workspace state, broadcasts to swarm
groups, and writes its own MCP config so spawned agent CLIs join the
same Ruflo brain.

The 4 bugs filed during the v1.1.1 dogfood are closed:
BUG-V1.1.1-01 (launch_pane fires but doesn't spawn) ✓
BUG-V1.1.1-02 (Sigma can't enumerate active sessions)   ✓
BUG-V1.1.1-03 (inter-agent broadcast inert)             ✓
BUG-V1.1.1-04 (Ruflo MCP not auto-connected)            ✓

What's new
==========

1. Tool dispatch parity
-----------------------

`runClaudeCliTurn.ts:routeToolUse()` extended with a `dispatchTool`
callback. Every `tool_use` block the Claude CLI emits is now routed to
`controller.invokeTool(name, args)`, the result captured, and a
`tool_result` envelope written back to the CLI's stdin:

* `tool_use_id` matches exactly (no orphan results)
* `is_error: true` flag on dispatcher exceptions
* Stringified content (the safer common-denominator shape across CLI
  versions)
* Serialised write queue so multiple tool_use blocks in one envelope
  go to stdin in order without race
* 30 s `withTimeout` for slow tools (`create_swarm` provisions
  worktrees; up to ~10 s typical)

The CLI then continues its turn with the actual host result in scope.
No more "Sigma traced the intent but nothing happened."

2. Live list_* tools
--------------------

Three new tools read the in-memory registries (NOT the DB):

* `list_active_sessions(workspaceId?)` — every live PTY pane with
  provider, agentKey, swarmId, paneIndex
* `list_swarms(workspaceId?)` — open swarms with roster + status
* `list_workspaces()` — every known workspace with active flag

`system-prompt.ts` trimmed: the stale recent-files + open-swarms blob
is gone; the prompt now says "call the list_* tools when you need
live state." Saves ~300 prompt tokens, makes Sigma's answers accurate
to the moment the question was asked.

3. MCP autowrite
----------------

New `app/src/main/core/workspaces/mcp-autowrite.ts`. On workspace open,
SigmaLink writes/merges three config files:

* claude code: project-local `<root>/.mcp.json`
* codex: global `~/.codex/config.toml` `[mcp_servers.ruflo]`
* gemini: global `~/.gemini/settings.json` mcpServers block

Each entry points the spawned agent CLI at the shared
`<root>/.claude-flow/` state dir, so every claude/codex/gemini pane that
boots inside a SigmaLink workspace converges on ONE Ruflo brain. Each
agent runs its own stdio MCP client (~50ms boot overhead), but they
all share memory, patterns, and learning on disk.

The autowrite is idempotent and refuses to clobber user-customised
`ruflo` entries (only writes if `command === "npx"` or no entry
exists). TOML round-trip preserves comments. Toggle in Settings →
Ruflo, kv key `ruflo.autowriteMcp` (default ON).

4. Mailbox group broadcast (the v1.1.1 dogfood bug)
----------------------------------------------------

`mailbox.ts:expandRecipient` correctly resolves `@all`,
`@coordinators`, `@builders`, `@scouts`, `@reviewers` to concrete
agent keys BEFORE invoking the PTY pane-echo closure. Operator
broadcasts now reach every recipient in the target swarm — and only
that swarm (the rc3 cross-swarm-leak fix is preserved).

5. SONA trajectory wrap
-----------------------

`runClaudeCliTurn.ts` now calls `trajectoryStart` on every CLI turn,
`trajectoryStep` per tool dispatch, `trajectoryEnd` with verdict on
success / failure / cancel. Fail-soft when Ruflo is unavailable
(missing supervisor never breaks a turn). Sigma Assistant accumulates
cross-session learning from its own tool-call outcomes via the
existing ReasoningBank pipeline — the same learning loop Claude (the
CLI) benefits from through the per-prompt [INTELLIGENCE] hook.

Upgrade path
============

Drop the new SigmaLink.app into /Applications. Existing kv state +
workspaces survive — only additions are the new canDo keys and the
optional `ruflo.autowriteMcp` toggle (default ON). First time you
open a workspace after upgrading, MCP config files are written for
the three agent CLIs if installed.
