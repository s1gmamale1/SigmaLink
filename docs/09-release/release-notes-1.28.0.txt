SigmaLink v1.28.0 — 2026-05-26
==============================

R-2 — native Cursor CLI provider. The last item on the post-roadmap backlog.

cursor-agent is now a FIRST-CLASS SigmaLink provider — not a generic shell pane.
It appears in the provider pickers, gets a worktree-isolated pane, session
resume, Ruflo MCP auto-bind, a model catalog, and the pane info bar: the same
treatment as Claude / Codex / Gemini / Kimi / OpenCode.

Built by one worktree-isolated Opus lane against the empirically-verified
cursor-agent v2026.05.24 contract (the CLI is installed locally) -> full gate
in main. No rpc-router change (provider plumbing is data-driven).


Added
-----

- `cursor` provider (shared/providers.ts):
    * command: cursor-agent
    * non-interactive panes: `-p` with `--trust` (headless floor; no-op in
      interactive panes) + conditional `--force` auto-approve (gated on
      autoApprove, mirrors claude/codex escalation)
    * resume: `--resume <id>` / `--continue` (universal-fallback like claude;
      not in DISK_SCAN_PROVIDERS — cursor has no documented JSONL session scan)
    * oneshot: `-p {prompt}` (substitutes in both launcher + swarm spawn paths)
    * models: sonnet-4 (default), gpt-5
    * install: curl https://cursor.com/install -fsS | bash
    * picked up automatically by listVisibleProviders / listDetectable
      (placed before the internal `shell` sentinel)

- Ruflo MCP auto-bind for Cursor: the autobind writer now also writes
  <workspace>/.cursor/mcp.json (same mcpServers shape as claude's .mcp.json;
  stdio + HTTP-daemon modes, idempotent merge, refuses operator-managed
  entries). Reuses writeJsonMcpFile verbatim.

- Cursor pane-splash model label + resume-arg wiring; new provider-registry
  test + resume/spawn-arg/MCP-autowrite cursor coverage.


Not in scope (tracked follow-up)
--------------------------------

- Skill provider-compat fan-out is NOT yet extended to Cursor. The skills
  ProviderTarget is a fixed claude|codex|gemini enum with an exhaustive
  never-check; cursor's on-disk skill/command layout is unverified. Adding it
  is a separate change (decide cursor's skill dir, extend ProviderTarget +
  targetDirFor + the fan-out + the renderer badge maps).

Operator smoke
--------------

1. Install: curl https://cursor.com/install -fsS | bash  (or the in-app
   provider install modal).
2. Auth: export CURSOR_API_KEY=…  OR  cursor-agent login  (once).
3. In SigmaLink, open the workspace launcher — "Cursor" now appears in the
   provider matrix. Launch a pane with an initial prompt -> spawns
   `cursor-agent --trust -p "<prompt>"` in the pane's worktree. Resume
   re-opens via `cursor-agent --continue`.
4. (Optional) MCP: confirm <workspace>/.cursor/mcp.json was written with the
   `ruflo` server; verify in-pane via `cursor-agent mcp list`.
