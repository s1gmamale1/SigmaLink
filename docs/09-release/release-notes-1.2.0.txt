SigmaLink v1.2.0 — Windows platform port

Windows 10/11 (x64) is now a peer release surface to macOS arm64.
NSIS installer ships from CI on every tag push, with a PowerShell
one-liner installer that mirrors the macOS curl-bash UX. Voice on
Windows routes through the Chromium Web Speech API; native SAPI5
binding is deferred to v1.3+. Code-signing is deferred indefinitely;
SmartScreen workarounds are documented inside the installer.

Why v1.2.0 exists
=================

The macOS stream stabilised at v1.1.11 (196/196 tests, lint 0/0,
native voice exceptions cleaned up). Platform reach is the next
growth lever — Windows is the largest installed dev workstation base
SigmaLink wasn't shipping to. v1.2.0 lands the platform without
regressing macOS.

What's new
==========

**NSIS installer for Windows 10/11 x64**
`.github/workflows/release-windows.yml` (70 LOC) builds on every
`v*` tag push (and `workflow_dispatch`) on `windows-latest`,
rebuilds native modules (`better-sqlite3`, `node-pty`) against
Electron 30 for x64, packs via `pnpm electron:pack:win`, and uploads
`SigmaLink-Setup-<version>.exe` to the GitHub Release via
`softprops/action-gh-release@v2`. Permissions `contents: write`;
concurrency group `release-windows-${{ github.ref }}` with
`cancel-in-progress: false`.

**PowerShell one-liner install (parity with curl-bash macOS)**
`app/scripts/install-windows.ps1` (234 lines / ~180 LOC) detects
AMD64, fetches the latest release tag (or a pinned tag via
`-Version v1.2.0`), downloads the EXE to `$env:TEMP`, calls
`Unblock-File` to strip the Mark-of-the-Web (`Zone.Identifier` ADS),
runs the NSIS installer, and cleans up unless `-KeepInstaller` is
passed. `-Quiet` forwards NSIS `/S`. PowerShell 5+ gate at entry.

**SmartScreen workaround documentation**
`app/build/nsis/README — First launch.txt` (72 lines) is surfaced
inside the installer via `nsis.license` (semantically odd but
functional; v1.2.1 polish replaces with a custom NSH page).
Documents the two recovery paths for users who download the EXE
manually instead of via the PowerShell flow:
- Option A: SmartScreen dialog → "More info" → "Run anyway"
- Option B: right-click EXE → Properties → check "Unblock" → OK

**Cascadia Mono terminal font on Windows**
`app/src/renderer/features/command-room/Terminal.tsx:112` prepends
`"Cascadia Mono"` to the xterm font stack ahead of Consolas/Menlo/
Monaco. Cascadia ships with Windows 11 and is installable on
Windows 10. macOS and Linux fall through to their existing stacks.

**Platform-aware Voice tab**
`app/src/renderer/features/settings/VoiceTab.tsx` reads
`window.sigma.platform` (new in `electron/preload.ts`) and switches
copy on non-darwin: `NATIVE_ENGINE_LABEL` becomes "Web Speech API
(Chromium, requires internet)"; `NATIVE_ENGINE_AVAILABLE` is `false`
so the native-engine radio is disabled; the diagnostics indicator
dot is grey neutral instead of red error ("expected state on
Windows", not "your voice is broken").

How to install
==============

macOS Apple Silicon (unchanged):

  curl -fsSL https://raw.githubusercontent.com/s1gmamale1/SigmaLink/main/app/scripts/install-macos.sh | bash

Windows 10/11 x64 (new):

  iex (irm https://raw.githubusercontent.com/s1gmamale1/SigmaLink/main/app/scripts/install-windows.ps1)

Or manual download from
https://github.com/s1gmamale1/SigmaLink/releases/latest — pick the
`SigmaLink-Setup-<version>.exe` asset on Windows or the
`SigmaLink-<version>-arm64.dmg` on macOS.

What's NOT in this release
==========================

* Native Windows SAPI5 voice binding — Web Speech API only;
  offline + always-on capture is v1.3+ work.
* EV/OV Authenticode signing — $300-700/yr; deferred indefinitely.
  SmartScreen workarounds documented in the in-installer README.
* `windowsControlsOverlay` frameless chrome — v1.2.0 uses native
  Windows frame + 140px Breadcrumb right-pad. WCO is v1.3+ polish.
* Linux AppImage + .deb test gating — built from the same source
  but no CI runner and no smoke; not on the supported platform list.
* `Split` + `Minimise` pane functions — still placeholder.
* Provider Registry Cleanup (BridgeCode + Cursor + Shell + Aider +
  Continue out, Kimi in as first-class) — pushed to v1.1.12 again.
* Kimi audit warning sweep (~100 findings across 20+ feature
  files) — v1.1.12.
* Microsoft Store / WinGet distribution — after EV cert.
* Windows auto-update — after signing.

Known issues
============

* **SmartScreen first-run warning on Windows** — fresh users who
  download the EXE manually (not via the PowerShell installer) will
  see SmartScreen on first launch. The in-installer
  `README — First launch.txt` documents the two recoveries:
  "More info → Run anyway" or right-click → Properties → Unblock.
  SmartScreen reputation is per-binary-hash, so every release
  re-warms reputation from zero.

* **`nsis.license` page shows the welcome README behind an
  "I accept" radio gate** — cosmetic; v1.2.1 polish target. The
  field semantically expects a license agreement and renders the
  text behind a forced accept gate. We're abusing it for the
  SmartScreen explainer. v1.2.1 replaces with a custom MUI2
  informational page via `nsis.include`.

* **Web Speech API requires internet on Windows** — Chromium
  delegates speech recognition to a Google cloud STT call.
  Air-gapped Windows users have no voice path. v1.3+ unblock via
  native SAPI5 binding.

* **No Windows auto-update** — users re-run the PowerShell
  installer to upgrade. Renderer toggle for auto-update stays
  opt-in and macOS-only.

* All v1.1.x macOS known issues unchanged (no Apple Developer ID,
  Porcupine wake-word licensing, Gemini pane resume).

Distribution
============

First dual-platform release. arm64 macOS DMG + x64 Windows NSIS EXE:

  # macOS
  curl -fsSL https://raw.githubusercontent.com/s1gmamale1/SigmaLink/main/app/scripts/install-macos.sh | bash

  # Windows
  iex (irm https://raw.githubusercontent.com/s1gmamale1/SigmaLink/main/app/scripts/install-windows.ps1)

  # Pin v1.2.0 explicitly on macOS:
  curl -fsSL https://raw.githubusercontent.com/s1gmamale1/SigmaLink/main/app/scripts/install-macos.sh | bash -s v1.2.0

  # Pin v1.2.0 explicitly on Windows:
  iex "& { $(irm https://raw.githubusercontent.com/s1gmamale1/SigmaLink/main/app/scripts/install-windows.ps1) } -Version v1.2.0"

Upgrade path
============

macOS users: drop the new SigmaLink.app into /Applications; existing
kv state + workspaces + agent sessions survive.

Windows users: first-time install via the PowerShell one-liner above
(or manual EXE download). No prior Windows release to migrate from.
