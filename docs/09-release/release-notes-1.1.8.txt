SigmaLink v1.1.8 ‚Äî Optimization release

A 5-agent parallel swarm landed the highest-impact non-breaking wins
from a 3-investigator Phase-1 scan. Zero behavioural changes. Zero
broken contracts. Cold boot is roughly 60% faster, the test suite is
fully green again, the lint baseline drops by 28, and the global
state module finally splits under the 500-line/file rule.

Highlights
==========

* **Main bundle: 97.57 ‚Üí 38.26 KB gzip (-61%, -59 KB)**.
  10 rooms now `React.lazy()`-loaded; `CommandRoom` stays eager. Cold
  paint is dramatically faster because the renderer no longer ships
  ~8.5 KLOC of room code (BridgeRoom 1790, OperatorRoom 1712,
  MemoryRoom 1136, SkillsRoom 1100, plus 6 others) before the user
  has picked a room.

* **PTY event fan-out fixed**.
  `pty:data` events used to register 32 `eventOn` listeners on a
  16-pane workspace (Terminal + PaneSplash, per pane), each
  matching sessionId by string compare and dropping 99% of events.
  v1.1.8 introduces `renderer/lib/pty-data-bus.ts` ‚Äî one
  `eventOn('pty:data')` listener that routes by sessionId through
  a `Map<string, Set<Listener>>`. 9 new bus tests; pure renderer
  change, IPC + main-process untouched.

* **6 failing main-process tests recovered: 108/114 ‚Üí 128/128**.
  `mailbox.test.ts` and `tools.test.ts` were hitting the
  better-sqlite3 NMV 123 ‚Üî 147 mismatch (Electron's ABI vs host
  Node 26's ABI). Both migrated to the canonical
  `vi.mock('../db/client')` + `fakeDb()` pattern already used by
  `session-restore.test.ts` and `manager.test.ts`. New shared
  `src/test-utils/db-fake.ts` consolidates the shim. Plus a new
  `factory.test.ts` covering `paneIndex` derivation + the 20-agent
  cap that v1.1.4 CommandRoom + tools dispatch both depend on.

* **state.tsx 996 ‚Üí 553 LOC** + 3 new sibling modules
  (`state.types.ts` 157, `state.reducer.ts` 316, `state.hook.ts`
  19) ‚Äî pattern mirrors the v1.1.4 `rooms-menu-items.ts` /
  `workspaces-summary.ts` precedent. All consumer imports keep
  resolving via `state.tsx` re-exports; no caller changes needed.
  The 553-LOC residual is the irreducible provider + IPC-listener
  effects (cannot be split without breaking React lifecycle
  cohesion ‚Äî that's a v1.1.9 custom-hooks refactor).

* **3 stub schemas in `rpc/schemas.ts` promoted to real zod**.
  `panes.resume`, `swarms.addAgent`, `skills.verifyForWorkspace`
  no longer use `{ input: any, output: any }`. Shapes drawn from
  the actual controller return types; `VALIDATION_MODE` stays
  `'warn'` (dev-only console output, non-breaking).
  Closes the v1.1.4 plan item.

* **Dead code wipe**: `src/lib/utils.ts` lost `parseAnsi` (which
  owned ALL 19 `no-control-regex` errors), `mockPTYBridge`,
  `generateId`, `formatDuration` ‚Äî all confirmed zero-caller.
  Orphan `PTYBridge` type removed from `src/types/index.ts`.

* **.data.ts sibling split for 7 shadcn UI files + RightRailContext**
  ‚Äî kills 7 `react-refresh/only-export-components` errors using
  the codebase's own pattern. Variants/types move next door; the
  component file imports + re-exports. Consumers updated where
  the lint rule still flagged re-exports (alert-dialog,
  pagination, calendar, toggle-group, RightRailSwitcher,
  RightRail). Sidebar.tsx variants also extracted as a bonus 8th
  split.

Coordination
============

Phase 1 ‚Äî 3 parallel investigators:
* `perf-investigator` ‚Äî bundle / React / IPC / SQLite / PTY survey
* `quality-investigator` ‚Äî file sizes, lint errors, stub schemas,
  dead code, type-safety holes
* `test-investigator` ‚Äî NMV-mismatch fix path + coverage gaps

Phase 2 ‚Äî 5 parallel coders (all NON-OVERLAPPING file scopes):
* `quality-lint-fixer` ‚Äî dead utils.ts + 3 schemas + 7 .data.ts splits
* `perf-bundle-lazy` ‚Äî React.lazy rooms + BrowserRoom latch
* `perf-ptybus` ‚Äî renderer pty-data-bus + Terminal/PaneSplash migration
* `test-nmv-fixer` ‚Äî vi.mock pattern + shared fakeDb + factory.test.ts
* `refactor-state-split` ‚Äî state.tsx ‚Üí 4 sibling modules

Each Phase-2 coder used `sparc:optimizer` / `sparc:tester` /
`analysis:performance-bottlenecks` skills and verified their own
gates before SHIP. One parallel-agent integration issue surfaced
(quality-lint-fixer's `.data.ts` split for `button.tsx` initially
broke `alert-dialog.tsx`'s `buttonVariants` import) and was
resolved within the same dispatch by quality-lint-fixer updating
the four consumer files (alert-dialog, pagination, calendar,
toggle-group). Lead verified aggregate gates green before tagging.
