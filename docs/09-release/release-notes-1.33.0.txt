Fixed
-----

- SF-9  `+Pane` button broken + Yolo/Bypass toggle overhanging the grid.
    SF-8 B3 had wrapped the compact `+Pane` toolbar button in a flex-col stack
    with a PERMANENT amber Yolo card below it. In the horizontal command-room
    toolbar that stretched the button full-width and the card overhung the
    panes (operator screenshot). Reverted the wrapper to the known-good
    `relative flex items-center gap-2` row and MOVED the Yolo/Bypass toggle
    INTO the `+Pane` dropdown menu (appears when the menu opens). The Launcher's
    own launch-form toggle is unchanged.


Added
-----

- SF-10  Assign a CLI label to a normal terminal.
    Click a pane's provider name in the header to tag it with the CLI actually
    running in it — e.g. a plain `shell` pane you launched `cursor-agent` in now
    shows as "Cursor" with that provider's colour. A "Reset to <real>" item
    clears it.

    DISPLAY-ONLY: the session's real provider_id is untouched, so spawn /
    resume / MCP autobind / the model catalogue / the drag payload all keep
    using the real provider. Only the header name + colour change.

    Persisted on agent_sessions.display_provider_id (migration 0025, nullable;
    added to the cross-machine sync allowlist) so the label survives restart /
    resume. New panes.setDisplayProvider RPC (broadcasts so the header
    re-renders).
